# 产品功能灵感备忘

> 记录时间：2026-05-16  
> 目的：保存后续可拆成 roadmap / GitHub issues / 版本计划的核心功能想法。

## 产品方向

价投合伙人不应只停留在“单只股票报告生成器”，更有潜力升级成一个本地 A 股研究工作流：

- 从单票分析扩展到自选股、组合、周报。
- 从 AI 观点扩展到可追溯数据、估值锚、同业比较和财报变化。
- 从一次性报告扩展到长期能力圈档案和研究记录。

## 核心功能候选

### 1. 自选股批量体检

允许用户输入一组股票，生成总览表：

| 字段 | 示例 |
|---|---|
| 股票 | `600519 贵州茅台` |
| 裁判结论 | `worth_research / skip / out_of_competence` |
| PE/PB 分位 | `PE 21% / PB 38%` |
| 质量指标 | `ROE / 毛利率 / 现金流匹配度` |
| 股息率 | `4.50%` |
| 主要风险 | `投资端踩雷 / 现金流失配 / 定价权弱` |
| 下一步 | `继续研究 / 等价格 / 放弃` |

价值：

- 比单只股票分析更适合日常使用。
- 很适合 README 展示和传播。
- 可以作为后续 Watchlist 周报的基础。

### 2. Watchlist 周报

给自选股生成每周摘要：

- 本周价格和估值变化。
- PE/PB 分位是否进入低估区。
- 分红、财报、公告事件。
- 哪些股票从 `skip` 变成 `worth_research`。
- 哪些股票风险项恶化。

价值：

- 提升留存。
- 让工具从“偶尔生成报告”变成“长期研究助手”。

### 3. 估值区间与安全边际面板

把估值锚产品化：

- 当前 PE / PB。
- 10 年中位数。
- 低估 / 合理 / 高估区间。
- 股息率锚。
- 用户自定义目标收益率。
- 当前价格距离安全边际区间的距离。

价值：

- 把 AI 报告变成可操作的研究面板。
- 强化项目的价值投资定位。

### 4. 反向质询模式

报告生成后，用户可以继续追问：

- 为什么不是好生意？
- 最大看错风险是什么？
- 如果股价再跌 30%，结论会变吗？
- 这份报告里最薄弱的证据是什么？
- 哪些关键数据缺失会改变结论？

约束：

- 只能基于当前报告、DataPack 和 reviewer 结果回答。
- 不允许模型自由补事实。

价值：

- 从单向报告变成真正的 copilot。
- 让用户更容易理解结论背后的假设。

### 5. 公司能力圈档案

用本地 SQLite 保存历史研究记录：

- 公司和行业。
- 商业模式摘要。
- 上次分析结论。
- 本次分析结论。
- 哪些部分看不懂。
- 常见 `out_of_competence` 行业。
- 用户备注和手动标签。

价值：

- 建立长期研究资产。
- 帮用户知道自己的能力圈边界。
- 可扩展到持仓周报和研究日志。

### 6. 财报变动雷达

对比最近两期或五期核心指标：

- ROE。
- 毛利率。
- 净利率。
- 经营现金流 / 净利润。
- 资产负债率。
- 分红。
- 营收和利润增速。

输出：

- 指标趋势表。
- 恶化项红色标记。
- “这次财报最值得追问的 3 个问题”。

价值：

- 数据含量高，能降低“只是 AI 写作文”的观感。
- 适合和批量体检、周报组合。

### 7. 同业比较页

基于 `industry-compare` 做 UI 和报告增强：

- 目标公司 vs 行业中位数。
- PE / PB / ROE / 股息率对比。
- 行业内估值排名。
- 行业内质量排名。
- “它贵是因为更好，还是单纯更贵？”

价值：

- 价值投资用户非常关心。
- 当前数据边车已有基础能力，落地成本相对可控。

### 8. 报告问答

对已生成报告做本地 Q&A：

- 这份报告为什么给 price FAIL？
- 引用了哪些原始数据？
- 哪些结论来自巴菲特，哪些来自段永平？
- 有哪些已知未知项？
- reviewer 认为哪里最不可靠？

约束：

- 只基于 `payload.json`、报告正文、judge、reviewer 输出回答。
- 问答结果要标注引用来源。

价值：

- 提升报告复用价值。
- 强化可追溯和反幻觉定位。

### 9. 公告 / 事件接入

优先接公告，谨慎接新闻：

- 财报发布。
- 分红方案。
- 回购。
- 大股东减持。
- 监管问询。
- 重大诉讼或处罚。

价值：

- 让周报和 Watchlist 更有现实感。
- 公告比新闻噪音低，更适合价投工具。

### 10. 投资检查清单模板

把报告转成可编辑 checklist：

- 是否理解商业模式。
- 是否有持续 ROE。
- 利润是否有现金流支持。
- 是否有分红纪律。
- 是否低于历史估值中位。
- 是否存在一票否决风险。
- 用户自己的备注和结论。

价值：

- 把 AI 输出变成用户自己的研究记录。
- 有助于形成“少做交易”的产品心智。

## 建议路线图

### 第一阶段：增强产品感

1. 自选股批量体检。
2. 财报变动雷达。
3. 同业比较页。

### 第二阶段：提升留存

1. Watchlist 周报。
2. 能力圈档案 SQLite。
3. 报告问答。

### 第三阶段：拉开差异化

1. 公告事件接入。
2. 反向质询模式。
3. 自定义投资检查清单。

## 当前最推荐优先做

优先做 **自选股批量体检**。

理由：

- 对用户价值最直观。
- 最适合开源项目传播。
- 可以复用已有 `buildDataPack`、大师分析、judge 和报告渲染链路。
- 后续 Watchlist 周报、能力圈档案、财报雷达都可以接在它后面。

宣传语候选：

> 把你的自选股丢进去，AI 先帮你筛掉一半不值得研究的。
